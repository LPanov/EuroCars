**Project Title: E-Commerce Auto Parts Platform**

Overview:
This project is a fully functional, full-stack e-commerce web application designed to showcase technical proficiency in the Java/Spring ecosystem and MVC architecture. It provides users with a seamless shopping experience for auto parts, from browsing the catalog to placing an order.

**Key Features**
 - Product Catalog Management: Allows users to browse a comprehensive list of auto parts.
 - Robust Filtering: Implemented advanced server-side logic to filter products by two critical criteria:
 - Car Model (e.g., Audi A3, Ford F-150) and Part category

Getting Started
**Prerequisites**
 - To run this project locally, you will need:
 - Java Development Kit (JDK) [17+]
 - Apache Maven [3.8+]
 - MySQL Server
 - An IDE (e.g., IntelliJ IDEA, Eclipse)

**Installation**
Clone the Repository:
   _ git clone [Your Repository URL]
     cd [your-project-name] __

Database Setup:
 _ spring.datasource.url=jdbc:mysql://localhost:3306/[your_database_name]
  spring.datasource.username=[your_username]
  spring.datasource.password=[your_password] _

Run the Application:
  _ mvn clean install
    mvn spring-boot:run _
    
Access the Application: 
    Open your web browser and navigate to http://localhost:8080/ (or the port specified in your configuration)

## ðŸ› ï¸ Database Initialization and Setup

> **IMPORTANT:** This sequence must be followed for the *initial* setup to ensure the application starts with required reference data.

1.  **Schema Generation:**
    * Start the `cart-svc` and the main application for the first time to automatically generate all required database tables.
    * User with email 'admin@admin.com' and password 'Admin123' will be generated automatically
    * Stop the application after the tables are created.

2.  **Reference Data Population:**
    * Locate the file `table_creations.sql` in the main folder of the project.
    * Run the insert statements inside this file. This process populates the core reference tables: `brands`, `models`, `engines`, `categories`, and `manufacturers`.

3.  **Application Data Seeding:**
    * Start the application again. The `PartInit` component will run, generating a sample `Part` record for testing.
